Hello! This is Actn.io

Open Source Backend as a Service

Read and write data, query your data without limits in real time. Set hooks as backend jobs, create front end modules with HTML5

Demo, an Example App

Demo app is already up and running on http://demo.actn.io and can be managed at http://demo.core.actn.io. Signin with demo@actn.io and password

Let's say you need to collect signatures for your online petition. Very basic scenerio can be using a web form to collect and save data into database. To achive that simple goal with Actn.io you need;

  • 1. Client to communicate with Cors API
  • 2. Model to validate and capture data properly
  • 3. Template to render petition form

If you already have a website or page 1. and 2. steps are enough to go

1. Client

You can create many clients as you like through Core API interface. Each client is dedicated to only one domain.

Client has two properties;

  • domain the domain which sends cross domain request
  • acl an array of access control list patterns for certain http verbs and paths. Eg: *, GET:/signatures, POST:/users, ...

Client for demo.actn.io domain

As you can see there is an apikey property belongs to Client. Use it to include javascript client into your pages.

<script src="http://demo.api.actn.io/connect?apikey=1b5f986d2136c22e775c519630b638dc"></script>

You can make server side http requests as well, but for that you need to authorize your requests by adding apikey and secret into request headers. This credentials reveal just after creating a new client and you need to save them since they aren't available after that event.

That's it, from that point you can call the actn.io cors api from your javascript code. We will learn how to make http later, but now let's create a Model to play with.

2. Model

Like Client, no limit on models too. Have a look on Signature Model which is already created for Demo App.

Model properties are;

  • name name of the model, cannot be change after creation and must be unique
  • schema JSON schema for structure and validations
  • indexes array of indexes
  • hooks dictionary of hooks

3. Template

Template allows you to create http accessible files like html pages, stylesheets or javacripts. It uses tilt gem internally but supports only erb as a templating language at the moment. You can manage templates here

Template properties are;

  • filename filename of the template, must be end with .erb, eg: index.html.erb, style.css.erb
  • layout layout filename to render template within, "true" if the template itself is layout
  • data_bind data binding options to fetch data before rendering template, the data will be available as data local variable
